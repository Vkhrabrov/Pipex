# Pipex

**Pipex** — это проект из школы 42, который демонстрирует основы межпроцессного взаимодействия и перенаправления ввода/вывода в Unix-подобных системах. Проект нацелен на создание функционала, аналогичного работе пайпов (pipes) в shell, где вывод одной команды передается на вход другой.

---

## Описание проекта

Проект реализует поведение следующей shell-команды:

```bash
< infile cmd1 | cmd2 > outfile
В процессе работы программа должна выполнить следующие действия:

Открытие файла для чтения:
Программа открывает файл infile для получения входных данных.

Выполнение первой команды:
Данные из infile передаются в первую команду cmd1.

Передача данных через пайп:
Вывод команды cmd1 направляется через созданный пайп на вход второй команды cmd2.

Запись результата в файл:
Вывод команды cmd2 записывается в файл outfile.

Ключевые концепции и технологии
Системные вызовы:

fork() — создание нового процесса.
pipe() — создание канала для межпроцессного взаимодействия.
dup2() — перенаправление файловых дескрипторов для корректного связывания ввода/вывода.
execve() — запуск внешних программ (команд) в дочернем процессе.
Работа с файлами:

Открытие файлов для чтения (infile) и записи (outfile).
Корректное закрытие файловых дескрипторов после завершения работы.
Обработка ошибок:

Проверка успешности выполнения системных вызовов.
Вывод информативных сообщений об ошибках для диагностики проблем.
Пример использования
Допустим, у вас есть следующие команды:

cmd1 — команда, которая обрабатывает входные данные.
cmd2 — команда, которая принимает вывод cmd1 и производит дальнейшую обработку.
Запуск программы выглядит следующим образом:

bash
Копировать
./pipex infile "cmd1" "cmd2" outfile
Где:

infile — файл, содержащий входные данные.
"cmd1" — первая команда для обработки данных.
"cmd2" — вторая команда для дальнейшей обработки.
outfile — файл, куда записывается конечный результат.
Зачем нужен проект Pipex?
Изучение системного программирования:
Проект позволяет освоить работу с ключевыми системными вызовами Unix, такими как fork(), pipe(), dup2() и execve(), что является фундаментальным для разработки системного ПО.

Понимание межпроцессного взаимодействия:
Реализация пайпов помогает лучше понять, как процессы могут обмениваться данными между собой.

Работа с файловыми дескрипторами:
Проект демонстрирует важность грамотного управления файловыми дескрипторами и обработки ошибок, что критично для стабильной работы программ в Unix-среде.

Заключение
Проект Pipex — это практическое задание, которое не только углубляет понимание основ Unix-систем, но и развивает навыки работы с межпроцессным взаимодействием, файловой системой и системными вызовами. Освоив этот проект, вы получите ценный опыт, который пригодится при разработке сложных программных решений и в повседневной работе с системным программированием.
